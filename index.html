HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • H1, H2, H3, etc designate the level of the heading.
  • ul is bullet points and ol is numbers, li is the designation under ul and ol (creates the bullet point and numbers)

CCS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • ALWAYS PULL ORIGIN MAIN BEFORE DOING ANYTHING. ON EVERY SINGLE BRANCH.
  • git status: checks out what breanch we are currently on
  • git checkout: changes to a certain branch
  • git checkout -b branch-name: creates a new branch name and switches to it
  • Make sure to use git checkout main and then use git pull origin main to make sure everything is up to date after committing and pushing.

JavaScript

  • Console.log is a way of putting things into the console, which is like a mini terminal.
  • Function is very similar to in python; however, it uses {} instead of :.
  • Nearly everything ends in ;
  • Alert makes a popup.
  • () => indicates an anonymous function. This basically is just saying it is an unnamed function that just executes.
  • Var vs. Let vs. Const = Basically variable is the old school useage. It is globally scoped or functionally scoped, depending on where it is defined. The issue with it is that we can change it in teh same scope without running into errors. Let is "block scoped", which means that it only exists within {}. So, if we say let greeter = "Hello" and then in another function {let greeter = "No"}. Then depending where we call, greeter will be "Hello" or "No". If we did this same thing with var, it will always be "No" if it meets the conditions. Const on the other hand cannot be updated or re-declared. Const is also block scoped.
